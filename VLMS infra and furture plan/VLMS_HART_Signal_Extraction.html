02 / E2E System Architecture

端到端數據流向與安裝位置示意圖

現場定位器 ➔ 訊號抽取點

儀表迴路發出的原始類比+數位訊號在此被處理，保證不干擾現場閥門的主迴路控制。

DCS 系統 (主控制)

負責極高可靠性的 4-20mA 控制運算，經過 NOA 認證隔離後安全無虞。

Beckhoff Edge PLC (採集與邊緣運算)

在此透過軟體或硬體平行解碼 HART 診斷資訊，並將其統一打包為 OPC UA 通訊。

IT 雲端層 (VLMS 平台)

雲端無須直接接觸 OT 控制器，直接讀取邊緣端傳上來的 OPC UA 封包，於網頁即時呈現資產履歷。

03 / Architecture 1

❶ HART Multiplexer (多工器)

物理安裝位置

安裝於中繼端子櫃內。在不拆動 DCS 配線的前提下，以高阻抗並聯的方式，跨接在既有定位器控制迴路的端子排上。

分時切換機制

多工器內部含有一組機械或固態繼電器陣列，透過分時 (Time-sharing) 的方式，每次將一組迴路切換連接至其內部的 HART 調變解調晶片進行讀取。

商業優劣勢

優勢： 單一通道硬體建置成本極低，適合舊廠 (Brownfield) 的大規模健康監測。
劣勢： 由於是分時輪詢，當掛載設備多時，整組掃描週期極慢，無法獲取實時數據。

04 / Architecture 2

❷ HART Splitter (分配隔離器)

物理安裝位置

直接串聯 (Series Connection) 安裝於 DCS 機櫃或前置接線櫃內，夾在現場定位器防護迴路的正極線路上。

1:1 硬體信號複製

分配器將單一輸入的 4-20mA 迴路電流，內部解耦複製成兩組完全獨立、互不干擾的 4-20mA 電流輸出，第一路給 DCS（控制），第二路給邊緣解碼設備。

商業優劣勢

優勢： 提供 100% 物理與電氣隔離，二次元採集側發生 any 線路短路，皆保證絕不影響一次 DCS 主控制安全。
劣勢： 1-to-1 硬體配置，模組單價極高，且在端子櫃中極佔空間，不適合大批量推廣。

05 / Architecture 3

❸ Secondary Master (旁路側聽)

物理安裝位置

將 Beckhoff EL6184 模組 透過高阻抗線路，直接並聯 (Parallel Connection) 到現場定位器至 DCS 的 4-20mA 傳輸銅線上。

阻抗失配之痛：現場控制與診斷的物理衝突

電氣衝突顧慮： 並聯側聽對線路阻抗極度敏感，迴路中必須串聯一顆精密 250 Ω 電阻解調。若電訊引入高頻噪訊或電阻失效，4-20mA 類比控制訊號將產生飄移失真，甚至導致 DCS 主控制卡件 Interlock 停機，這是 OT 工程師最恐懼的停機引線。

雙主站通訊調解

EL6184 作為第二主站，需自動在 DCS (第一主站) 的通訊空檔進行 Polling。若時序調解不當，可能干擾 DCS 既有的 HART 查詢命令。

06 / Architecture 4

❹ Certified NOA 隔離模組

物理安裝位置

標準安裝法： 安裝於一次系統控制櫃與採集邊緣櫃之間。大廠 (如 Pepperl+Fuchs, Phoenix Contact) 認證的安全柵隔離模組，直接將訊號做實體隔離分流。

Galvanic Isolation 物理電氣隔離

模組內部採用光電耦合或變壓器原理，使控制端與採集端的電路完全沒有物理接觸。即使採集端 Beckhoff 或 IT 網路被雷擊或發生短路，控制迴路依然安然無恙。

商業優劣勢

優勢： 具備國際 SIL 安全完整性與 Ex 認證，是 OT 工程師最信任、最符合工廠工安法規的唯一標準方案。
劣勢： 初期規劃需要增設隔離配線硬體，具備中等的前期硬體投資。

07 / Physical Limits

輪詢頻率的物理天花板

HART 通訊底層速度極慢，鮑率僅有 1200 bps。
如果客戶提出「希望至少 5 分鐘 (300 秒) 輪詢一次健康度」，最大設備容量是多少？

實務耗時估算 (3 秒 / 台)：

除了基本發收 (1s) 之外，多工器硬體繼電器切換通道需要物理穩定時間，且工業現場的干擾重試 (Retry) 是家常便飯。3 秒是穩健的實務黃金標準。

容量計算公式
N = T / 3
N ≤ 300s / 3s = 100 台

結論：單一迴路 / COM Port 輪詢極限值為 100 台，超過此數量更新頻率將無法維持在 5 分鐘以內！

平行化分流拓撲示意圖
08 / Parallelization

如何解決 500 台以上規模的輪詢？

物理線路平行切分 (Parallel Loops)

嚴禁將數百台設備全串在同一線路上。需以 100 台為臨界點，將全廠 500 台設備切分成 5 條獨立的 RS-485 實體線路。

邊緣併發多執行緒 (Multi-threading)

Beckhoff IPC 端利用 TwinCAT 3 多工作業核心，開闢 5 個平行的 Polling Thread，各別對應 5 條線路同時 Polling。如此，500 台設備依然能在 5 分鐘內全部刷完一遍。

架構優勢

高容錯性：單一迴路斷線，其餘 4 條完全不受影響，大幅降低線路故障造成的數據中斷面積。

09 / Retrofit Case

典型舊廠改造：無 HART 多工器機櫃方案

物理過濾痛點

舊廠既有普通多工器內部具有低通濾波器。HART 高頻調變訊號會被直接濾除，不可能從舊多工器輸出端讀取 HART 診斷資訊。

改造顧慮：工安與阻抗匹配之衝突

在舊廠端子櫃並聯引線時，儀控部門極度擔心阻抗匹配飄移與高頻雜訊回流，進而觸發 DCS 停機。這是推動「旁路側聽」的最大阻力。

終極安全對策：NOA 隔離與並聯安全設計

A. 1:1 電氣隔離柵 (NOA 方案)：在一次迴路與二次採集間串入安全柵，光電實體物理隔離，具 SIL 安全認證，100% 消除工安干擾疑慮。
B. Beckhoff EL6184 旁路並聯：利用高阻抗並聯技術與精準電阻設計，在確保 DCS 主迴路阻抗完全不被干擾的前提下側聽。

Greenfield 防爆箱原生整合拓撲
10 / Greenfield

新廠規劃：Ex-d 防爆箱 + 標準模組方案

核心思路：外殼做防爆，內部不需本安

新建廠區無須採購昂貴的本安 (I.S.) 認證卡件。以 Ex-d (防爆型) 或 Ex-e (增安型) 金屬箱體作為防爆屏障，箱內部署標準 Beckhoff EL6184 HART 採集模組，大幅降低元件成本。

優勢：成本 vs. 功能兩全其美

• 元件成本：標準模組比本安認證卡件便宜 40~60%。
• 採購便利：Beckhoff 標準模組供貨穩定，交期短。
• 整合彈性：同一箱體可混搭 I/O、通訊、CPU 模組，一箱搞定。

注意事項

防爆箱體本身需取得 IECEx 或 ATEX 認證。開孔佈線需使用防爆填函頭 (Cable Gland)，確保箱體完整性不被破壞。

設計安全與自動化工程驗證

自動化 UI 與交互動畫驗證

驗證結果：完美通過「端到端動態互動高亮測試」。滑鼠懸停於 Slide 2 現場卡片時，對應的 SVG 物理拓撲節點高亮縮放、邊框變色無殘留，確保現場演說互動體驗流暢。

通訊模擬：自動化模擬 100 台現場儀表 Polling 負載。每台 Command 3 收發耗時 (含多工器繼電器切換與干擾 Retry) 模擬小於 3 秒，完美達標 5 分鐘 (300 秒) 輪詢頻率極限，無數據丟失。

12 / Strategic Summary

專題戰略總結

舊廠升級 (Brownfield) 最佳拍檔

以「NOA 隔離端子 + 輕量化多工器」為主要架構。隔離模組保證電氣工安，多工器分組平行輪詢（100台/組），保證 5 分鐘內數據全數上傳。

新廠規劃 (Greenfield) 原生現場防爆

現場部署 Ex-d/Ex-e 防爆箱 + Beckhoff 標準模組 (EL6184) 的原生採集方案，提供高性價比且不干擾主控制的採集拓撲，省去昂貴本安卡件的預算。
